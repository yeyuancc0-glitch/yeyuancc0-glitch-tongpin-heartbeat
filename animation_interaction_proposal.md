# 同频跳动 - 全局动效与微交互设计方案 (V0.1B 核心交互)

既然布局先维持原样，我们完全可以把精力聚焦在“细节的质感”上。基于“情绪胶囊”的治愈系品牌基因，这套动效设计的核心目标是：**“物理真实感”、“情感共鸣”与“细腻的跟手性”**。

动画不应只是为了好看而加的视觉装饰，它必须是用户交互的自然延伸。

---

## 一、 核心设计原则与底层逻辑

### 1. 物理弹簧模型 (Spring Physics)
全面抛弃传统的线性 (Linear) 或简单的缓入缓出 (Ease-in-out) 动画，改用**基于物理属性的弹簧模型（Spring Animation）**。
这能赋予 UI 元素“质量”、“张力”和“阻尼”，让它们看起来像是真实世界中的物体。无论是按钮按下还是弹窗出现，都带有自然的弹性与生机。

### 2. 触觉反馈先行 (Haptics First)
视觉动效必须与设备的马达震动（Haptic Feedback）紧密绑定。在执行任何改变状态的微交互时，配合不同强度的震动，让数字屏幕上的操作转化为真实的指尖触碰感，极大地提升 App 的“高级感”。

### 3. 完全跟手与可打断 (Gesture-Driven & Interruptible)
所有涉及手势的动画（如拖拽、下拉关闭）必须是实时跟随手指的。更重要的是，它们必须是**随时可打断的**——动画播放到一半时，用户再次触摸或滑动，元素应基于当前的初速度和平滑曲线立即响应新指令，绝不出现卡顿或强制等待。

---

## 二、 全局基础微交互规范

### 1. 弹性呼吸点击 (Bouncy Press)
- **适用场景**：所有主按钮、图标按钮、快捷互动贴纸。
- **动效表现**：手指按下瞬间，元素迅速且柔和地缩小至 `0.95` 比例；手指抬起或移开时，通过弹簧函数迅速回弹至 `1.0`，并伴随肉眼仅能微弱感知的阻尼振荡余波。
- **触感配合**：按下瞬间触发极其轻微的震动反馈（`Light Impact`）。

### 2. 骨架屏光影过渡 (Breathing Skeletons)
- **适用场景**：网络请求等待、相册图片异步加载。
- **动效表现**：摒弃廉价的“菊花转圈”。使用与实际内容轮廓完全一致的占位底色卡片，并附带平缓的、类似人类缓慢呼吸频率的渐变光波扫描（Alpha 值在 `0.3` 到 `0.7` 之间周期浮动）。内容加载完成后，通过 Cross-fade (交叉淡入淡出) 丝滑切换，拒绝生硬跳变。

### 3. 错误拒绝抖动 (Error Shake)
- **适用场景**：表单校验失败、邀请码错误等拦截场景。
- **动效表现**：模拟人在“摇头拒绝”的物理动作，输入框发生快速的左右位移衰减抖动（位移轨迹：`+10px -> -10px -> +5px -> -5px -> 0`）。
- **触感配合**：伴随两次短促且强烈的错误震动提示（`Error Notification`），手指一震就知道出错了。

## 三、 核心业务模块动效设计

### 1. 底部导航栏 (Bottom Tab Bar) 的“黏性”切换
- **图标微动效**：点击非当前 Tab 时，被点击的图标先执行弹性下沉（Scale `0.8`），选中瞬间向上跃起并微微溢出原始大小（Scale `1.15`），最后弹簧回落。
- **背景光效追踪**：底部选中的高光/背景药丸块，在切换时不应是生硬跳转，而应该像液滴一样，以“黏滞流体”的方式平滑追踪滑过底部轨道，落入新的 Tab 槽位。

### 2. 相册大图的共享元素飞行 (Shared Element Transition)
- **打开照片**：在相册列表中点击一张缩略小图时，图片**物理上从原卡片位置剥离并“飞出”**，在飞行的半空中平滑拉伸至全屏比例，背景同时匀速变暗。
- **下滑抛弃手势 (Swipe to Dismiss)**：在全屏查看大图时，用户随时可以按住图片向下拖拽。拖拽时图片等比例缩小，周围透出背后的主页面；松手时，图片根据手指赋予的惯性和方向，精准“飞回”列表中的原始坐标（若速度足够，背景则瞬间点亮）。

### 3. 情绪胶囊投递的“抛物线涟漪”
- **投递飞出**：在首页选择快捷心情并点击“发送”时，对应的心情图标会化作一颗微小的“胶囊”，沿着计算好的贝塞尔曲线（抛物线）飞向上方的“此刻同频”头像区。
- **波纹扩散**：当胶囊命中头像的一瞬间，头像周围爆开一圈极其柔和的透明度波纹涟漪，象征情绪的成功传递。
- **触感配合**：命中瞬间触发清晰的成功震动反馈（`Success Notification`）。

### 4. 半屏弹窗的阻尼下拉 (Rubber Banding Bottom Sheet)
- **适用场景**：写留言、写信件的弹出面板。
- **越界阻尼感**：面板不仅可以下拉关闭，当用户按住面板顶部继续强行**往上拖拽**时，不应像死板的网页一样毫无反应。应提供“越界阻尼弹性”（Rubber Banding），面板跟随手指向上产生微小的拉伸形变，松手后像被橡皮筋拉住一样“噔”地回弹归位。

## 四、 技术落地建议（用于后续实施参考）

为确保这套高度依赖性能的动效不卡顿，建议使用：
1. **React Native Reanimated (v3)**：彻底剥离 JS 桥接，在 UI 线程直接运行所有核心弹簧计算与共享元素转场，确保 60/120FPS 的绝对丝滑。
2. **React Native Gesture Handler**：接管底层手势系统，与 Reanimated 无缝配合实现完美的跟手打断逻辑。
3. **expo-haptics**：在所有封装好的基础组件（Button, Card, BottomSheet）层面对接马达反馈。
