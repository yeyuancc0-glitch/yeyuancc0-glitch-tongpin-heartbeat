# V0.1A 验收标准

## 用户路径

1. 用户 A 注册登录并保存 profile。
2. 用户 A 创建邀请码。
3. 用户 B 注册登录并保存 profile。
4. 用户 B 输入邀请码。
5. 系统通过 `accept_pair_invite` 创建 `couples` 和两条 `couple_members`。
6. 双方进入情侣首页。
7. 双方可以查看恋爱天数、纪念日倒计时、今日分享状态、留言和日历事件。
8. 任一方解绑后，双方不能继续写入原 couple 数据。

## 功能验收

- 注册 / 登录可用。
- profiles 可创建和更新。
- 创建邀请码时不会提前创建空 couple。
- 接受邀请码时会校验双方都没有 active couple。
- 情侣首页展示双方资料。
- 今日分享支持创建和更新。
- 留言板支持创建和作者软删除。
- 基础日历事件支持创建和展示。
- RLS 拦截跨情侣读取和写入。

## 不验收内容

- 相册上传。
- 未来信 UI。
- 原生推送。
- AI 助手。
- 订阅支付。
- 完整审核后台。
